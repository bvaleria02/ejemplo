Guía de Laboratorio: Sesión 5

Profesor: Matthias Clein                                                                                                                     Fecha: 27-09-2021
La finalidad de la Sesión 5 del Laboratorio de Programación de Computadores es que los estudiantes repasen
y  pongan  en  práctica  los  contenidos  de  Elementos  Básicos  de  un  Programa.  Deberán  crear  una  carpeta
llamada laboratorio-5 y dentro escribir códigos en lenguaje de programación C (llamados actividad-X.c) que
resuelvan  los  problemas  planteados  en  las  siguientes  actividades.  Al  finalizar  los  ejercicios,  deberá  subir  la
carpeta a su repositorio de GitHub.

Actividad 1: Repaso
• El programa debe solicitarle al usuario ingresar dos números
  y posteriormente mostrar por pantalla
  el resultado de la suma de ambos números.
• El programa debe ser amigable con el usuario.

Actividad 2: Código ASCII de un carácter.
• El programa debe solicitarle al usuario ingresar un carácter
  cualquiera para luego imprimir dicho carácter por pantalla
  seguido del valor que le corresponde en el código ASCII.
• El programa debe ser amigable con el usuario.

Actividad 3: Multiplicación de variables float.
• El programa debe solicitarle al usuario ingresar dos números
  reales en una misma línea de la forma “X.X , Y.Y”, para
  posteriormente mostrar por pantalla el producto de la
  multiplicación entre ambos.
• El programa debe ser amigable con el usuario.

Actividad 4: Cálculo del perímetro y área de un círculo.
• El programa debe solicitarle al usuario ingresar el radio
  de una circunferencia para luego imprimir por pantalla los
  valores de su perímetro y su área.
• El programa debe ser amigable con el usuario.
• Recuerde definir la variable PI.

Actividad 5: Cálculo de la última nota del curso.
• Escriba un programa que le permita al usuario calcular la nota
  mínima que debe tener en la Prueba 2  para  aprobar  un  curso.
  Tome  como  referencia  las  ponderaciones  de  las  evaluaciones
  de  esta asignatura.  El  programa  deberá  pedirle  al  usuario
  que  ingrese  las  notas  obtenidas  en  las  otras evaluaciones
  para posteriormente imprimir por pantalla la nota necesaria para
  aprobar el curso.
• El programa debe ser amigable con el usuario.
• El programa debe tener una “pantalla de inicio".
